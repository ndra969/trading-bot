# Pre-commit Hooks Setup Guide

## Installation

1. Install pre-commit (if not already installed):
```bash
uv sync --extra dev
```

2. Install the git hooks:
```bash
uv run pre-commit install
```

## What the hooks do

The pre-commit hooks will automatically run before each commit:

1. **Code Formatting** (Black)
   - Automatically formats Python code
   - Line length: 100 characters

2. **Linting** (Ruff)
   - Fast Python linter
   - Automatically fixes issues where possible

3. **Type Checking** (mypy)
   - Static type checking
   - Excludes tests and alembic directories

4. **Unit Tests** (pytest)
   - Runs all unit tests in `tests/unit/`
   - Ensures all tests pass before commit

5. **Coverage Checks**
   - **Overall Coverage**: Minimum 85% for entire codebase
   - **Risk Management**: Minimum 95% coverage (critical component)
   - **Position Management**: Minimum 95% coverage (critical component)

6. **Foundation Engine Tests**
   - Specifically runs Phase 5 foundation engine tests
   - Ensures Phase 5 implementation is working

## Running hooks manually

Run all hooks on all files:
```bash
uv run pre-commit run --all-files
```

Run specific hook:
```bash
uv run pre-commit run pytest-all --all-files
uv run pre-commit run pytest-coverage-overall --all-files
```

## Skipping hooks (not recommended)

If you need to skip hooks for a commit (emergency only):
```bash
git commit --no-verify -m "Emergency commit"
```

**Warning**: Only skip hooks in emergencies. The hooks ensure code quality and prevent bugs.

## Coverage Requirements

- **Overall**: 85% minimum
- **Critical Components** (risk, position): 95% minimum
- **New Features**: 100% coverage required

If coverage is below threshold, commit will be blocked. Fix tests to increase coverage before committing.
